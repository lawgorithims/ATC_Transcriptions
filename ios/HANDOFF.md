# CommSight — Session Handoff

Pick up the **CommSight** iOS app (on-device ATC transcription) in a fresh session without losing
context. Pairs with `RECOVERY.md` (rebuild the ship box) and the Claude **memory notes** (which hold
the secret IDs and box details — this repo is **PUBLIC**, so nothing secret lives here).

> ⚠️ **Public repo.** Do NOT commit secrets, App Store Connect key IDs, the `.p8`, or the build-box
> host/IP. Those live in the memory notes named below and on the build box.

## 1. What this is

- **CommSight** — universal iPhone/iPad SwiftUI app, fully on-device ATC radio transcription.
  Fine-tuned Whisper via **WhisperKit** (CoreML/ANE) + a two-tier corrector (deterministic + an
  optional CPU llama.cpp / Apple-Foundation LLM) + RAG context. Sources: internet LiveATC feed,
  device mic, USB audio, replay demo.
- Repo: **`lawgorithims/ATC_Transcriptions`** (local dir is `ATC_Transcribe`). Work on **`main`**.
  All app code is under `ios/ATCTranscribe/`. The Xcode project is generated by **xcodegen** from
  `ios/project.yml` (no checked-in `.xcodeproj`).
- App Store Connect app **CommSight**, bundle `com.flycommsight.atctranscribe`. **Builds 1–8 shipped
  to TestFlight on 2026-06-27**; current `main` is the build-8 state.

## 3. Build / test / ship loop (the actual cadence used all session)

The build box (a headless Scaleway M4 Mac, `User m1`) has Xcode, the models, and the vendored
`llama.xcframework`. Local edits on Windows; **nothing compiles on Windows**.

**Iterate (sync changed files → build/test):**
```bash
# from ios/ on the Windows box, tar the files you changed to the M4 (no --delete; never touch
# Resources/Models or Vendor):
tar czf - ATCTranscribe/UI/Foo.swift ... | ssh m1@<host> 'cd ~/ATC_Transcribe/ios && tar xzf -'
# on the M4: regen + test against the BOOTED simulator (id from `xcrun simctl list devices booted`):
export PATH="$HOME/.xcodegen/xcodegen/bin:$PATH"
xcodegen generate
xcodebuild -project ATCTranscribe.xcodeproj -scheme ATCTranscribe \
  -destination 'platform=iOS Simulator,id=<BOOTED_SIM_ID>' \
  -derivedDataPath /tmp/dd-commsight CODE_SIGNING_ALLOWED=NO test
```
- Adding a **new .swift file** requires `xcodegen generate` before `xcodebuild` (globs at gen time).
- `git reset --hard origin/main` on the M4 is the safe way to put it on an exact commit before
  shipping (Resources/Models + Vendor are untracked/ignored and survive the reset).

**Ship to TestFlight (outward-facing — only when the user asks):**
```bash
# on the M4, in ios/, with the ASC env from the memory note + the NEXT build number:
ASC_KEY_ID=... ASC_ISSUER_ID=... ASC_KEY_PATH=/Users/m1/AuthKey_*.p8 TEAM_ID=... \
BUILD_NUMBER=9 bash Tools/ship_testflight.sh
```
`ship_testflight.sh` is the **headless-correct** recipe (archive UNSIGNED → sign-for-distribution at
export). `Tools/testflight.sh` does NOT work over SSH. **LEAN=1** (default) ships model-less; the app
downloads models on first launch from HuggingFace (`SingularityUS/atc-whisperkit`).

**Version gotcha (fixed build 3+):** `project.yml` info binds `CFBundleVersion=$(CURRENT_PROJECT_VERSION)`
and `CFBundleShortVersionString=$(MARKETING_VERSION)` (=1.0). Without that, xcodegen baked a literal
`1` and uploads collided. Builds ship as `1.0 (N)`.

## 4. Working practices that worked well

- **Cadence:** edit → sync → `xcodebuild test` on M4 → fix → commit to `main` → `reset --hard` M4 →
  `ship_testflight.sh BUILD_NUMBER=N`. Each batch = one build.
- **Adversarial review before every ship.** Run a `Workflow` that fans reviewers across dimensions
  (concurrency, SwiftUI, DSP, regressions) then adversarially verifies each finding against the code.
  This caught real HIGH bugs every batch (audio-session leaks, an AVAudioEngine re-attach crash, a
  mic data race, a hallucination-filter resurrection, a per-segment decode blowup). **Keep doing it.**
- **Tests:** unit + XCUITest on the Simulator. `test1_onboardingGateAndSkip` XCTSkips when a model is
  bundled (the gate only shows on lean builds) — that 1 skip is expected. The **neural path** (real
  ANE) is validated by the macOS `ATCKitProbe` (`Tools/probe.sh`), NOT the Simulator (no ANE there).

## 5. Architecture map (key files)

- `Engine/LivePipeline.swift` — actor: source chunks → VAD → **diarize** → preprocess → transcribe →
  correct → `TranscriptRecord`; runtime setters (`setCorrector/LLM/Gate/Squelch/Diarization`).
- `Engine/TranscriptionSession.swift` — @MainActor ObservableObject wrapper; owns audio-session
  teardown on stop/natural-end; `inputLevel`.
- `UI/AppModel.swift` — the view-model (sources, model switching, standby, squelch, diarization,
  showDebug, widgets, onboarding gate). `start()/beginCapture()` (mic permission + session activate).
- `Audio/` — `AudioSource` (mic/USB/replay), `StreamAudioSource` (LiveATC MP3 decode),
  `AudioSessionManager`, `AudioMonitor` (live-feed speaker monitor), `VADSegmenter`, **`Diarizer`**.
- `Core/` — `ATCCorrector` (deterministic + **HallucinationFilter** + chain), `RepetitionCollapse`,
  `LocalLLMCorrector`/`FoundationModelsCorrector`, `ConfidenceGate`, RAG (`ATCKnowledge*`).
- `UI/` — `ConsoleView` (top bar, controls, input meter, standby banner, `SquelchControls`),
  `TranscriptView` (rows + speaker chips + amber correction highlight), `SidebarView` (customizable
  widgets incl. `DiagnosticsCard`), `SettingsSheet`, `Theme` (blue accent), `DeviceLoad` (CPU/thermal).
- `Download/` — `ModelCatalog`, `ModelDownloadManager`, `OnboardingDownloadView` (first-run gate).

## 7. OPEN LOOPS — what a fresh session should pick up

**Device-only validations the user still needs to confirm (can't be tested headless):**
1. **Microphone actually captures** on device (build 5 created the engine after the session is
   active + added a no-audio watchdog). If still silent, the watchdog message + the exact on-screen
   error are the clues.
2. **Live-feed speaker monitor** — audio comes out the speakers (build 5).
3. **Battery** while idle/backgrounded (the session-leak fixes + standby).
4. **Diarization quality** on a real feed — does it put ATC/aircraft on separate lines, or
   over-split / mislabel? Thresholds in `Diarizer.swift` are easy to tune; toggle in Settings → Speakers.

**Known limits / candidate next work:**
- **Diarization can't separate simultaneous talkers (overtalk).** The heuristic v1 splits on PTT
  gaps + acoustic clustering. The "very robust" upgrade = a dedicated **speaker-embedding CoreML
  model** (project on the scale of the Whisper conversion: pick/convert a model, host on HF, wire a
  download + per-segment embedding + clustering). Scope it if the user wants overtalk handling.
- HallucinationFilter blocklist is just `["no call of"]` — extend `Diarizer`… (actually
  `Core/ATCCorrector.swift` `HallucinationFilter.phrases`) as new Whisper phantom phrases surface.
- `noisereduce` preprocessing stage from the Python is still not ported (see `AudioPreprocessor`).
